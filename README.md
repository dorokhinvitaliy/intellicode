# IntelliCode Fabric

**AI-ассистент для глубокой разработки с контекстным пониманием всего проекта.**

IntelliCode Fabric — это не просто чат-бот для VS Code. Это продвинутая автономная AI-среда, которая объединяет гибридную систему поиска с генерацией ответов (RAG) и функциональными мультиагентными возможностями. Расширение умеет не только объяснять код, но и *изучать* проект, *создавать*, *редактировать*, *удалять* файлы и *выполнять* консольные команды — и всё это с вашего прямого одобрения через систему маркеров и уведомлений.


## Требования для запуска

- **Visual Studio Code:** Версия `1.80.0` или выше.
- **Node.js:** Версия `18.x` или выше (используется для компиляции TypeScript-кода и запуска скриптов среды).
- **Провайдер API:** Доступ к одному из поддерживаемых сервисов (OpenAI, Anthropic и т.д.) и действительный API-ключ. *По умолчанию поддерживаются LLM-модели, способные генерировать сложные многокомпонентные ответы (например, gpt-4o, claude-3.5-sonnet).*


## Как установить и запустить

1. **Клонируйте репозиторий проекта** и перейдите в папку с расширением:
   ```bash
   cd extension
   ```

2. **Установите зависимости:**
   ```bash
   npm install
   ```

3. **Скомпилируйте TypeScript код в JS:**
   ```bash
   npm run compile
   ```

4. **Запуск расширения в VS Code:**
   - Откройте папку проекта `extension` в окне VS Code.
   - Нажмите `F5` на клавиатуре для запуска сессии отладки (Extension Development Host).
   - В открывшемся новом экспериментальном окне VS Code у вас появится панель IntelliCode Fabric.

5. **Настройка API-ключа или подключение локальной модели:**
   - Перейдите в настройки тестовой среды VS Code (`Cmd + ,` или `Ctrl + `).
   - Найдите раздел **IntelliCode Fabric**.
   - Установите вашего провайдера (например, *openai*), саму модель (например, *gpt-4o*) и вставьте ваш персональный *API-ключ*.
   - Добавьте локальную модель, например, ollama

## User Flow

Главное взаимодействие происходит через боковую панель **IntelliCode Fabric Chat** и дерево **RAG Status**.

### Шаг 1: Индексация проекта
Прежде чем AI сможет понимать ваш код, его нужно "скормить" векторной базе.
- Откройте исходную папку вашего проекта в новом окне VS Code.
- Нажмите кнопку **"Индексировать проект"** на боковой панели (RAG Status).
- Запустится `ProjectIndexer`: он умным образом обойдёт ваши файлы (исключая `node_modules`, `.git`, бинарники), разобьет их на смысловые куски (чанки) и скачает им семантические эмбеддинги. Отслеживать статус кэширования можно в статистике: она покажет общее число чанков и файлов. Индексы изолированы для каждого нового воркспейса.

### Шаг 2: Взаимодействие и общение
Откройте вкладку **IntelliCode Fabric Chat**:
- **Обычный вопрос:** Напишите *"Как устроена авторизация?"*. Модель получит вопрос, найдет нужные чанки кода через гибридный векторный поиск и выдаст человекопонятный ответ *без попыток изменить код*.
- **Приказ:** Напишите *"Создай класс UserServices в папке front и запусти npm run dev"*. Модель:
  1. Напишет свое *"Решение"* во внутреннем блоке раздумий (`<thinking>`).
  2. Воспользуется `<<<CREATE_FILE>>>` маркером, вызвав интерфейс предварительного просмотра кода.
  3. VS Code создаст окошко предупреждений: *"AI хочет создать/отредактировать файл"*. После вашего подтверждения (кнопка `Approve`), файл будет сохранён.
  4. Модель использует маркер `<<<EXECUTE>>>`, предложив запустить команду. После вашего клика скрипт будет выполнен в консоли, а её вывод (лог сервера или ошибка) автоматически вернётся к ИИ для анализа.

### Шаг 3: Мульти-агентная автоматизация (Экспериментально)
Помимо стандартного чата, в коде заложена оркестрация агентов (`Analyst`, `Coder`, `Tester`, `Refactor`). Для их вызова:
1. Выделите нужный участок кода (или весь файл) в редакторе VS Code
2. Вызовите меню команд (Ctrl+Shift+P / Cmd+Shift+P) команды **"IntelliCode Fabric: Refactor"** (или воспользуйтесь горячей клавишей). Так же можно воспользоваться контекстным меню. 
3. Введите инструкцию: агенты проанализируют выделенный код и предложат улучшения прямо через Inline-редактор, где вы можете принять или отклонить изменения.
*Примечание: в данный момент flow агентов находится в процессе активной разработки.*


## Архитектура решения

IntelliCode Fabric состоит из набора независимых модулей, каждый из которых решает свою конкретную задачу. Вот как код проходит от нажатия Enter до создания файла в вашем проекте:

1. **`extension.ts` (Точка входа)**
   Главный контроллер. Инициализирует LLM, базу данных (VectorStore) и регистрирует провайдеры для Webview (интерфейс чата) и TreeView (панель RAG-статуса).

2. **`ChatHandler.ts` (Мозг общения)**
   Отвечает за весь диалог. Перед походом в LLM обогащает ваш запрос контекстом (RAG) и просит AI соблюдать жесткие системные промпты (например, обязывает использовать маркеры \`<<<EDIT_FILE>>>\`). 

3. **`SidebarChatProvider.ts` (Интерфейс)**
   Управляет Webview окном (HTML/JS вкладкой внутри VS Code). 
   - Имеет встроенный "санитайзер" Markdown, умеющий отображать кастомные графические элементы (заголовки, кнопки копирования).
   - *Перехватывает* сгенерированные AI маркеры на лету, прячет их от пользователя и передает функциональным обработчикам.

4. **`FileOperationsHandler.ts` (Система безопасности файлов)**
   Единственный класс, имеющий право трогать вашу файловую систему. Когда он получает команды `CREATE`, `EDIT` или `EXECUTE`, он парсит пути, *защищает систему от абсолютных путей-двойников* (`resolveFilePath`), а затем высвечивает UI-предупреждение "Approve / Reject" (с вызовом `vscode.diff` для редактирований). Поддерживает управление длиноживущими терминальными процессами (dev-серверами).

5. **`ProjectIndexer.ts` & `VectorStore.ts` (RAG-система)**
   - **`ProjectIndexer`:** Занимается сканированием (учитывая `excludePatterns` и маски), нарезкой текстов (chunking) и умным хешированием (caching — пропускает файлы, которые не менялись, экономя токены и время).
   - **`VectorStore`:** Хранит чанки в JSON формате (`vector_store_HASH.json`). Отвечает за **Гибридный поиск**, где итоговый Score считается по формуле: *Семантическое совпадение + Ключевые слова + Совпадение путей + Буст важных файлов*.

6. **`LLMClient.ts`**
   Низкоуровневая обёртка над OpenAI/Anthropic API, умеющая обращаться за обычными текстами (`chatStream`) и за векторами (`createEmbedding`).


IntelliCode Fabric превращает ваш код в живую базу знаний и дает вам полноценного виртуального разработчика-напарника, работающего прозрачно и только с вашего разрешения.
